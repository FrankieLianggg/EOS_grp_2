## Project Overview
This project focuses on converting the Prestige Cars database into a normalized and well-structured database. The work includes User Defined Types, normalized table creation, constraints, data integrity improvement, ERD/PDM documentation, PowerPoint preparation, and final submission organization.

## Main Files
- create_UDT.sql: creates reusable User Defined Types and required schemas.
- create_tables.sql: creates normalized tables using the UDTs.
- ERD/PDM files: show database design, keys, relationships, and cardinality.
- Presentation files: explain project lifecycle and design decisions.
- Tracking file: shows group task progress and revision notes.

## UDT Work
The project uses reusable UDTs for keys, codes, names, addresses, money values, date/time values, and boolean fields. This improves consistency and makes the database easier to maintain.

## Normalized Tables
The normalized tables include SalesRegion, Country, Customer, Make, Model, Stock, Sales, and SalesDetails. The design separates data into logical subject areas and reduces duplication.

## Data Integrity
Primary keys, foreign keys, unique constraints, default constraints, and check constraints were used to improve data integrity and enforce business rules.

## Final Deliverables
The repository contains SQL scripts, ERD/PDM evidence, PowerPoint presentation, project tracking file, and final submission materials.
